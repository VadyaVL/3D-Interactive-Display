# 3D Interactive Display

[Завантажити повний код проекту!](https://drive.google.com/file/d/0B9uL21byGFsEWEJqLTNMMk9wUms/view?usp=sharing)

**3D Interactive Display** - система, що дозволяє за допомогою **Glyph** та **5DT Data Glove** здійснювати оберти моделі, яка відображеється на екрані планшету та проектується на 3D голографічну піраміду.

Для роботи системи необхідно:
* ПК чи ноутбук з web камерою та додтком-сервером;
* планшет із встановленим додатком для Android;
* 3D голографячна піраміда;
* гліф з рукавичкою віртуальної реальності;
* доступна WI-FI мережа.

Для початку роботи необхідно:
1. Наклеїти гліф на рукавичку та підключити її до ПК/ноутбука.
2. Запустити додаток **сервер**, або додаток **симулятор рукавички-сервер** на ПК/ноутбуці підключеному до доступної WI-FI мережі.
3. Налаштувати додаток на ПК/ноутбуці.
4. Запустити додаток на планшеті, який підключений до тієїж WI-FI мережі.
5. Розмістити 3D голографічну піраміду на планшеті.
6. Розпочати роботу з рукавичкою і сервер-додатком.

Рекомендації:
* після підключення рукавички для її калібрування - кисть руки стиснути в кулак;
* гліф має бути в полі видимості web камери.

Програмне забезпечення:
* **GloveServerNET**: MS Visual Studio 2013, Afroge.NET, Driver for 5 DT Glove.
* **Interactive_3D_Display**: JMonkeyEngine 3.0, niftyGUI, Android SDK.
* **Poster**: yEd graph editor.

### Рекомендації щодо подальшого вдосконалення проекту:
1. Як видно, при роботі з симулятором рукавички, модель обертається плавно. Проте з використанням рукавички і камери оберти відбуваються ривками. Це не залежить від мережі. Ривки відбуваються з-за повільного визначення/обчислення нових кутів положення гліфу. Для уникнення ривків можна спробувати:
* використати більш потужний ноутбук/ПК.
* написати функцію, що буде апроксимувати положення рукавички.
 
2. Можна розробити гру, де за допомогою камери та рукавички ми будемо переміщувати певну площину з отворами таким чином, щоб в отвори попадали падаючі елементи.
 
3. Також можна реалізувати, щось подібне до конструктора. Але це потребує реалізації переміщення моделі в просторі, що є не складним. Бібліотека Aforge.NET дає можливість легко отримати такі дані.
 
4. Можна реалізувати різні анімації, з якими ми зможемо взаємодіяти. Наприклад дотик рукою до води, удар по м’ячу, опади (дощ, сніг), і т. п. 
 
5. Необхідно повернути фрейм програми на 45 градусів. Зараз з цим є певні проблеми. Див. JMonkey ViewPort Rotate.
